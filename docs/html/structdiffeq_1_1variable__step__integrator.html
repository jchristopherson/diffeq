Attempts a single integration step.

Syntax
subroutine attempt_step( &
class(variable_step_integrator) this, &
class(ode_container) sys, &
real(real64) h, &
real(real64) x, &
real(real64) y(:), &
real(real64) yn(:), &
real(real64) en(:), &
optional real(real64) xprev(:), &
optional real(real64) yprev(:,:), &
optional real(real64) fprev(:,:), &
optional class(errors) err &
)
Parameters
[in,out]thisThe variable_step_integrator object.
[in]sysThe ode_container object containing the ODEs to integrate.
[in]hThe current step size.
[in]xThe current value of the independent variable.
[in]yAn N-element array containing the current values of the dependent variables.
[out]ynAn N-element array where the values of the dependent variables at x + h will be written.
[out]ysAN N-element array where the supplemental solution values at x + h will be written.
[in]xprevAn M-element array containing the previous M values of the independent variable where M is the order of the method. This is typically useful for multi-step methods. In single-step methods this parameter is not used.
[in]yprevAn M-by-NEQN array containing the previous M arrays of dependent variable values where M is the order of the method. This is typically useful for multi-step methods. In single-step methods this parameter is not used.
[in,out]Anoptional errors-based object that if provided can be used to retrieve information relating to any errors encountered during execution. If not provided, a default implementation of the errors class is used internally to provide error handling.

Syntax
real(real64) pure function compute_next_step_size( &
class(variable_step_integrator) this, &
real(real64) hn, &
real(real64) en, &
real(real64) enm1 &
)
Parameters
[in]thisThe variable_step_integrator object.
[in]hnThe current step size.
[in]enThe norm of the error for the current step size.
[in]enm1The norm of the error from the previous step size.
Returns
The new step size.
Remarks
The step size estimate makes use of a PI type controller such that \( h_{n+1} = f h_n \left( \frac{1}{e_n} \right)^{1/k} e_n^{\alpha} e_{n-1}^{\beta}\), where \( f \) is a safety factor, and \( k \) is the order of the integration method.

◆ step()

procedure, public diffeq::variable_step_integrator::step

Takes one integration step.

Syntax
subroutine step( &
class(variable_step_integrator) this, &
class(ode_container) sys, &
real(real64) x, &
real(real64) xmax, &
real(real64) y(:), &
real(real64) yn(:), &
optional real(real64) xprev(:), &
optional real(real64) yprev(:,:), &
optional real(real64) fprev(:,:), &
optional class(errors) err &
)
Parameters
[in,out]thisThe variable_step_integrator object.
[in]sysThe ode_container object containing the ODEs to integrate.
[in]xThe current value of the independent variable.
[in]xmaxThe upper integration limit.
[in]yAn N-element array containing the current values of the dependent variables.
[out]ynAn N-element array where the values of the dependent variables at x + h will be written.
[in]xprevAn M-element array containing the previous M values of the independent variable where M is the order of the method. This is typically useful for multi-step methods. In single-step methods this parameter is not used.
[in]yprevAn M-by-NEQN array containing the previous M arrays of dependent variable values where M is the order of the method. This is typically useful for multi-step methods. In single-step methods this parameter is not used.
[out]fprevAn M-by-NEQN array where the previous M function values are written. This is typically useful for multi-step methods. In single-step methods this parameter is not used.
[in,out]Anoptional errors-based object that if provided can be used to retrieve information relating to any errors encountered during execution. If not provided, a default implementation of the errors class is used internally to provide error handling. Possible errors and warning messages that may be encountered are as follows.
  • DIFFEQ_MEMORY_ALLOCATION_ERROR: Occurs if there is a memory allocation issue.
  • DIFFEQ_ARRAY_SIZE_ERROR: Occurs if yn is not the same size as y.
  • DIFFEQ_STEP_SIZE_TOO_SMALL_ERROR: Occurs if the step size becomes too small.
  • DIFFEQ_ITERATION_COUNT_EXCEEDED_ERROR: Occurs if the iteration count is exceeded for a single step.
  • DIFFEQ_NULL_POINTER_ERROR: Occurs if no ODE routine is defined.
